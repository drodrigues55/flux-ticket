# Agent System Rules & Instructions

Sempre que realizar modificações de código, fluxos ou interface no projeto **Flux Tickets**:
1. **Verificar Documentação Existente:** Identifique se existe documentação técnica relevante na pasta `docs/` (como guias de arquitetura, concorrência, design-system, ou ciclo de vida de tickets).
2. **Atualização do Walkthrough:** Continue registrando os detalhes técnicos das mudanças no `walkthrough.md` localizado na pasta de artifacts correspondente à sessão.
3. **Sincronização com Docs da Pasta `docs/`:** Se houver um arquivo `.md` relevante que documente o comportamento ou design do sistema que foi modificado, você DEVE atualizá-lo para refletir com exatidão a implementação real corrente (código, cores, regras de negócio ou fluxos). Não permita que a documentação em `docs/` fique defasada em relação às modificações feitas.
